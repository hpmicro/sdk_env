37.4. FULL_DUPLEX_SPI_SLAVE_POLLING

37.4.1. 概述

full_duplex_slave_polling示例工程展示了SPI作为master使用polling方法进行板与板之间全双工通信。 在这个示例工程中， 一个板子作为SPI master， 另一个板子作为SPI slave。 Master发送一定量数据到Slave并从Slave接收一定量的数据。

37.4.2. 硬件设置

将两个板子的 SPI引脚 相连。

两个板子必须相连双方GND进行共地，保证信号传输。

37.4.3. 注意

  • 仅支持SPI单线模式，不支持两线、四线模式。

  • 在polling模式下,作为从机，SPI总线的SCLK频率过高可能会导致FIFO来不及写入或读取导致overflow,在高sclk频率下推荐使用DMA方式。

  • 为保证正常传输，SPI的CS引脚建议加上4.7K的上拉电阻。

37.4.4. 运行现象

  • 先运行slave，再运行master

  • 当工程正确运行后，串口终端会输出如下信息：
