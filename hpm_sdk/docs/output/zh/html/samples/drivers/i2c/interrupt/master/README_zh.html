19.5. i2c_interrupt_master

19.5.1. 概述

i2c_interrupt_master示例工程展示了I2C在master模式使用interrupt方式接收和发送数据的功能。在这个示例工程中，I2C master先发送数据，之后再接收数据，并对比发送和接收到的数据。

19.5.2. 硬件设置

将两个板子的I2C引脚相连。

19.5.3. 运行现象

  • 先运行slave，再运行master

  • 当工程正确运行后，串口终端会输出如下信息：
